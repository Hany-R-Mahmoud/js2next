# Human Content Review Playbook

A designated reviewer must approve every packet and assessment set before publication.

## Lesson review

- Scope matches the approved curriculum map.
- Explanation is independent and original.
- No distinctive source-course project or wording is reproduced.
- Technical claims match official current documentation.
- Version-sensitive behavior is named.
- Examples are plausible, focused, and free of accidental dependencies.
- Prerequisites appear before dependent ideas.
- Terminology is consistent with the rest of the track.
- The lesson can be understood without the source video.
- Common mistakes are realistic and corrected constructively.

## MCQ review

- Exactly one unambiguously correct choice.
- Prompt maps to a stated objective.
- Distractors are plausible but clearly wrong for a knowledgeable learner.
- No grammar, length, or formatting clue reveals the answer.
- Code questions can be answered by reasoning without execution.
- Explanation teaches why the answer is correct.
- Wrong-choice feedback identifies the misconception where useful.
- Difficulty label is reasonable.
- No source-course trivia or project-specific memory is required.

## Publication action

1. Reviewer records name, date, and notes.
2. Packet moves from `draft` to `reviewed`.
3. `reviewStatus` moves to `approved`.
4. A separate publish action adds the exact version to the publication manifest.
5. Later changes create a new review requirement.
