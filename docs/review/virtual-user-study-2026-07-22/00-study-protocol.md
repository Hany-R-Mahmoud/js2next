# JS2Next Virtual-Learner UX Study

Target: `http://localhost:3000`.

This is a synthetic, browser-observed usability study. It does not represent real human research, regional network conditions, long-term retention, or full assistive-technology coverage.

## Cohort

- 20 independent personas: 7 beginner, 7 intermediate, 6 advanced.
- 14 MENA personas, 6 global personas.
- English UI tested by English-first and Arabic-first bilingual learners.
- Arabic findings classify comprehension, localization readiness, bidi/RTL behavior, and terminology separately.

## Session rules

1. Use the assigned `uNN.localhost:3000` origin to isolate local progress.
2. Do not inspect source code, prior reports, or other persona findings.
3. Use browser-visible state first; capture DOM/accessibility evidence and screenshots for important findings.
4. Complete shared navigation, search, learning, practice, check, review, progress, settings, and assessment flows.
5. Primary module: complete Learn → Practice → Check → Reflect and module assessment.
6. Secondary modules: inspect structure, complete one Learn, Practice, and Check interaction.
7. Give an immediate teach-back after the primary module.
8. Do not reset progress or mutate project files.

## Report contract

Record task, expectation, action, result, friction, evidence, confidence, and score (1–5) for:

- clarity, organization, navigation, level fit, comprehension, cognitive load;
- Learn/Practice/Check/Reflect flow;
- practice, assessment feedback, review, progress;
- English readability, Arabic readiness, accessibility, responsiveness;
- strongest positives, highest-impact problems, and continuation intent.

Severity: P0 blocked, P1 core task failure, P2 major friction, P3 polish.

## Evidence gate

Each report needs cold-entry, module, and end-state evidence. Every P0–P2 issue needs URL, reproduction steps, observed/expected result, and DOM/screenshot evidence when available.
