## Teach-back

JavaScript is an ECMAScript language implemented inside a host such as a browser or server runtime. HTML handles structure, CSS presentation, and JavaScript behavior. Primitive values have runtime types, while variables can refer to different types over time. Functions are easier to test when inputs and outputs are explicit, and larger behavior can be composed from smaller functions.

## Issues

### P1 — Module progress is not reflected after reopening

- URL: `http://u01.localhost:3000/learn/javascript/orientation-and-foundations`
- Steps: Complete all four topics and the module assessment at 83%; visit Progress; reopen the module.
- Observed: Progress showed 4/4 mastered and module complete, but the reopened module showed “0/4 required topics mastered,” all topics “Ready to learn,” and Practice/Check/Reflect locked.
- Expected: Reopened module should hydrate from saved progress and show the completed state.
- Evidence: Progress DOM showed `4/4 required topics mastered`, `Module review: 83%`; reopened module DOM showed `0/4 required topics mastered`.

### P1 — Visible module/topic links often did not navigate

- URLs: `/tracks/javascript`, module and topic routes.
- Steps: Clicked visible Module 1, topic, and “Start topic Check” links.
- Observed: URL remained on the parent page; only active styling changed. Following the visible href directly worked.
- Expected: Clicking a visible link should navigate normally.
- Evidence: Accessibility snapshots showed `[active]` on the link while URL remained unchanged.

### P2 — Arabic search returns no localized matches

- URL: `http://u01.localhost:3000/home`
- Input: `جافاسكريبت`
- Observed: “Showing 0 of 79 topics. No canonical topics match this search.”
- English control test: `JavaScript` returned 30 topics.
- Expected: Arabic aliases, transliteration, or a localization hint could improve Arabic-first discoverability.

### P2 — Assessment feedback order changed

- URL: Functions Working Together quiz.
- Observed: Pre-submit question order differed from post-submit feedback order, making answer tracking harder.
- Expected: Preserve question order or clearly explain sorting.
