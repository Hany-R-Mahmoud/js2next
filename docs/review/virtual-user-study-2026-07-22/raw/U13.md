## Severity-tagged issues

### P1 — Module stage state is inconsistent and blocks continuation

- JS-M10 assessment submitted at 100% and Progress showed `2/2`, `100%`, Complete.
- Reopening `/learn/javascript/modern-javascript-development` showed `0/2`, topics “Ready to learn,” and Practice/Check/Reflect locked.
- Expected: module page should reflect saved mastery and unlock module stages.
- Impact: the primary assigned module cannot reach its intended module workflow.

### P2 — Onboarding defaults do not match the learner persona

- Welcome screen showed “Beginner path”; Settings initially showed Beginner, Active, and reduced motion Off.
- Expected: Intermediate/Visual/reduced-motion preferences should be collected or correctly initialized.

### P2 — Arabic search has no localization support

- English `closure` returned one topic.
- Arabic `جافاسكريبت` returned `Showing 0 of 79 topics` and `No canonical topics match this search.`
- Expected: bilingual matching, transliteration/translation suggestions, or a clear supported-language explanation.

### P2 — Settings change unexpectedly altered Focus area

- Focus started as “Use the full map” and later showed “React fundamentals” without direct selection; it was manually restored.
- Expected: unrelated settings changes should not silently change focus.

### P2 — RTL/bidi behavior remains insufficiently verified

- Arabic reflection input was accepted and saved, but no full RTL layout or direction controls were available.

No P0 issues observed.

## Language and accessibility notes

- Interface content was English-only.
- Short explanations, hints, examples, and answer feedback supported English second-language comprehension.
- Arabic text entry worked safely in reflection; Arabic search did not find English topics.
- Skip links, landmarks, headings, progress bars, tabs, radios, switches, and labelled controls were present.
- Reduced motion could be enabled; full RTL/bidi validation was not reached.

## Confidence and limitations

Overall confidence is high for observed UI behavior, topic flows, and the Progress inconsistency; low-to-medium for RTL/bidi because full RTL was not available. R-M05 and N-M03 were partial, the cumulative assessment was not submitted, and no reset/destructive recovery was attempted.

## Continue decision

Do not continue the study path until the P1 module-progress synchronization issue is fixed. Topic-level learning and assessments are usable, but the primary module cannot complete its intended workflow through the UI.
